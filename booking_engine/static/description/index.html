Booking Engine Module

The Booking Engine Module is not part of Odoo standard but uses Odoo Studio and leverages lines of python code; therefore installing this module may increase your subscription.

Purpose

Booking Engine Module aims at enabling the management of stay offers.

It enables:

  • The definition of stay offers and supporting roles and resources;
  • The management of bookings from a dedicated app;
  • The guest management in terms of identity and stay offer unit;
  • The management of city taxes due to local authorities.

Stay Offer definition

The data model of the booking engine relies on standard models from Odoo Planning.
However, their relationships are tight in this configuration and deserve explanations.

Stay Offers

Stay Offers correspond to Products and have specific behavior due to their Is Stay Offer setting.
This additional setting allows to filter only the Stay Offers in the Accommodation App and supports some logic.

In order to book Resources when renting the corresponding Stay Offer, those Products plan Services with specific Roles, which is hidden to user to simplify the setup.
On Stay Offer creation, a Role is created with the same name and binded to the Stay Offer.

Additional specific configuration is that they may have Variants relying on different Attributes.
For each combination of Variant Creation set as Instantly, defined in their Attributes & Variants tab, they shall be associated a period and price in the Rental Prices tab.

Last but not least, Resources associated to the Stay Offer can be linked from the Resources tab.

Rental Periods

Rental Periods are available from the Rental App Configuration menu and define the unit of time for rental offers.

💡

Note that the Rental Period Nights is to be used for stay offers.


⚠️

Rental Periods defined with Nights as Unit enable to define Check-in and Check-out time of the day, which will be used to set the booking start and end time.

Roles

Roles make the bridge between the Product, the Stay Offers, and the Resources (rooms, houses, etc.)

Key aspect of the Booking Engine configuration is the Sync Shift and Rental Orders setting of the Roles.
It prevents double booking of Resources by preventing overlapping shifts for Resources associated with the Role.
This reflects both in the Accommodation App with the impossibility to confirm orders when booking would collide with existing ones as well as in the website where the calendar is computed according to resource availabilities.

⚠️

The roles are hidden to users in the setup of Stay Offers with Resources.
They are available from the Planning App.

Resources

Resources correspond to the Stay Offer units customer actually rent, such as rooms, houses, pitches, bungalows...

The Stay Offers fields allows to edit the binding of Resources to Stay Offers.
Updating this binding in practice updates the binding of Role and Resources, seamlessly for users.

A key aspect of the Booking Engine configuration is the Working Time of Resources which is set to Rental 24/7.

A custom field Occupant is provided to capture at first sight the currently renting customer in different views.

💡
Note that Resources can be associated to multiple Roles even though they have a default one.

Attributes

The Attributes associated to Variants enable to define Stay Offer variations, either as informative content, or impacting the Stay Offer pricing.

The Display Type of Attributes specifies how customers would select between two values from the e-Commerce website.

Variant Creation set as Never

This refers to informative only characteristics such as Free Wifi Included or Smoking Prohibited which can be displayed on the stay offer pages.

Variant Creation set as Instantly

In such case, the value combination of attributes lead to as many Stay Offer variations.
Examples for such Attribute are Number of Guests, Breakfast Included or not...

Each combination might lead to different rental price, to be defined from the Rental Prices tab of Stay Offers.

Guests Attribute

The Guests attribute is configured to capture the number of guests.
The number of adults and children can be fill in for each attribute value.

Those inputs are leveraged:

  • To compute the number of guests in the sale order lines.
  • To pre-fill the number of guests per room in the city tax widget.


Accommodation App

The Accommodation App, labelled per the industry name, provides access to the core of the booking engine and enables the booking management.

Booking Schedule

In this app users land on the booking Schedule where they access the Gantt view of their stay offer bookings for all their stay offer resources.
This view provides load per resource and day, and provides different interactive capabilities.

Searching a booking

User can use the search bar to quickly shortlist the customer bookings.

The list view is a good alternative when no bookings are setup during the currently display period.

Opening an Order

Clicking on a Shift opens a popup from which the Rental Order number is clickable to reach it.

Creating a new booking

User can drag and drop over a resource line a given period, which opens a modal to save a shift for this resource.
Clicking the New Order button will bring users to the order form where they can select a customer, cross check order details and confirm the booking.

Adding a stay offer booking to the last order

User can drag and drop over a resource line a given period, which opens a modal to save a shift for this resource.
Clicking the Add to Last Order button adds a line or updates the quantity of the corresponding stay offer to the last order.

💡

The last order is considered as last created order by user.

Changing Resource and Upgrading Offer

User can drag and drop an existing Shift to another Resource, including to a Resource of another Stay Offer / Role.
In the second case, this can be used for upgrades from a standard room to a deluxe room for example.

Changing Shift and Order dates

User can drag and drop an existing Shift from left to right to change the Shift and associated Order dates.

⚠️

Given such change acts on the Order dates, this as well waterfall on the different rented Products and their Shifts.
This means that all Shifts associated to this Order will change dates.


💡

The Sync Shifts and Rental Orders setting prevents this to happen in case the change collides with any other Shift.

Blocking a Resource

It may happen that a Resource is not available to rent in case of maintenance for instance.
In such cases, user can create a Shift on the corresponding resource line without creating a new Order or adding to the last Order.
Notes in such case might be valuable to make sure all employees are aligned about the purpose of the Shift.

Orders

The Orders menu provides a Kanban view of all rental Orders for Stay Offers.

They are grouped by Stage and at first focusing on today's order for check in and check out, while different filters allow users to efficiently search among them.


Guests

Guest management is handled in the module in two ways.

Guest identity

During check in, some countries require guest identity to be captured.

To support users with this duty, from the Order Guests tab, Contacts can be related to the order.
Opening a contact form allows to input guest identity details, which is reflected in the order.

Similarly to Orders listing Guests, Guest stays are listed from their contact form with a smart button at the top of the screen.

Ongoing rental

On top of their stays, as soon as a guest is checked in a room, the corresponding contact form displays the Resources checked in.
This enables to know at anytime where a guest is staying.

On the other hand, the Accommodation App provides the current guest occupant from the Resources.


City Tax Management

The City Tax are managed through a dedicated product, added to the Order as soon as the customer checks in.

At the end of the stay, at check out from the Order, a dedicated popup helps user to compute the amount of City Tax units due.
It summaries the bookings of the Order and their nights as well as an estimate of guest number based on the listed Guests in the Guests tab.

All fields are editable to compute the number of City Tax units to apply, which updates the City Tax product delivered quantity on save.


House Keeping Setting

⚠️

The House Keeping Setting is not part of Odoo standard but uses Odoo Studio and leverages lines of python code; therefore installing this setting may increase your subscription.


Purpose


House Keeping aims at enabling the management and reporting of house keeping to support accommodation industries: Hotel, Campsite and Guest House.

It enables:

  • The monitoring of room cleanness status with a dedicated board;
  • The assignation and execution of tasks for house keeping per rooms.

Tasks

House Keeping is handled through Tasks from a dedicated Project.
The house keeping tasks have:

  • An employee in charge of the cleaning work;
  • A resource to be cleaned among the stay offer resources;
  • The resource current occupancy, being vacant when the room is unoccupied, occupied when a guest has checked in, stayover when guest are staying, due out when the guests are expected to leave;
  • A cleaning instruction capturing the level of effort to spend, stayover when are staying overnight so standard cleaning is needed, or checkout when guests are leaving so in depth cleaning is expected;
  • A deadline for the day of the task;
  • A stage to capture the task completion, dirty when the task is to be done, in progress, clean when the cleaning work is complete and ready for approval, ready the cleaning work is approved.
Creation

At night, a scheduled action creates the house keeping tasks for resources requiring cleaning:

  • Resources with stayover guests
  • Resources with guests leaving
Assignation

After the task creation, assignation to employees relies on the employee planning for house keeping and the house keeping tasks.
Employees are assigned tasks as follow:

  • For the house keeping roles they are assigned to;
  • With cleaning points in proportion of their shift duration of the day.
Approval

The settings enable to define approvers who can check and approve cleaning is complete for each room.
House keeping tasks listed in the Clean stage can be approved by one of the approvers listed in the settings.

💡

When the approvers setting is not set, house keeping tasks set to Clean stage automatically go to the Ready stage and get closed.


Roles

House Keeping roles are planning roles allowing to assign house keeping shits to employees, and are available from the House Keeping menu.

With this setup, employees house keeping shifts can be planned as usual with Planning App.

Resource assignation

The house keeping roles are assigned as well to stay offer resources from the Accommodation App / Stay Offers / Roles menu.
In this list view, user can as well provide stay over cleaning points and checkout cleaning points.
Those points are used for the house keeping tasks assignation and capture that certain stay offers might take more time and effort to clean than others and that the stayover cleaning effort might be lesser than the checkout cleaning one.
The cleaning points is an abstract and relative scale.

💡

Pro tip to assign cleaning points:

  • Ideally use the modified Fibonacci suite for cleaning points: 1, 2, 3, 5, 8, 10, 15, 20, 30...
  • Set 1 point for the stayover cleaning of the simplest stay offer.
  • Assign points to all other configuration to all pre-existing assignation.

⚠️
Stay offer roles without house keeping role or with null cleaning points does not lead to house keeping task creation.


Board

House Keeping menu is exposed in the accommodation apps and gives access to the board.

The board gives an overview of all resources involved to support stay offers with visual information to capture at first sight the cleanness status.

  • The occupancy captures the presence of guests for this resource.
  • When a task is in progress for house keeping of this resource, the task name is provided as well as the assigned employee.
    The task stage is as well exposed.

Planning

House Keeping Planning is a replicate of the Planning App view with a focus on house keeping shifts.
