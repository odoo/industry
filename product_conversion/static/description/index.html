Product Conversion Module

To provide maximum flexibility during warehouse operations, a new Convert button is now available on transfer from the backend and from the barcode app.




Smart Visibility Logic

The Convert button appears only when specific criteria are met to ensure data integrity:

  • Inventory Presence: The transfer must not be a Receipt or Dropship. (Products must be physically in stock before they can be converted).

  • Storable Products: Only products with "Track Inventory" enabled are eligible.

  • Operational Status: The transfer must be in a state other than "Draft" or "Done."

The Conversion Interface

Clicking the button opens a guided pop-up containing:

  • Source Details: Displays the product and the specific "Pick From" (the quant), including the location and the selected Lot or Serial Number.

  • Quantity Validation: The system prevents converting more than the maximum available stock for that specific Quant.

  • Traceability Management:

    • Keep Lot/SN: If checked, the new product inherits the existing Lot/Serial Number (e.g., new "T-shirt (Quality Refurbished, Yellow)" retains the original "LOT1").

    • New Lot/SN: If "Keep" is unchecked, users can manually assign a new Lot/Serial Number.

    • Traceability Matching: The system intelligently filters "Convert Into" options based on tracing. For example, if you convert a Lot-tracked item and provide a new Lot number, the destination product must also support Lot tracking. But if you dont provide any Lot Number, only product without specific traceability will be eligible for convert into.

  • Destination Location: If you have more than 1 internal location set up on your inventory, you can choose the destination location as well.


Backend Execution & Traceability

Once clicking on "Convert" button an Automated Manufacturing Order (MO) is executed in the background:

  • Location Accuracy: The MO pulls from the exact "Pick From" location to ensure inventory balance.

  • Detailed Logging: A record of the conversion is automatically posted to the Chatter of both the Transfer and the Manufacturing Order.

  • History Preservation: By using a background MO, the Traceability Report remains unbroken. You can view the entire history of the new product, including its previous form.


Inventory Adjustment

During the inventory adjustment process, users can utilize the conversion feature by selecting a specific line item and triggering the 'Convert' action



Variant Control & Error Prevention

While the system allows broad product-to-product conversion, you can enforce strict rules using the "Convert Attribute" setting:

  • Restricted Mapping: If a product uses an attribute marked "Convert Attribute," the user is limited to selecting variants within that specific attribute group.

  • Error Prevention: This guardrail ensures users only convert products into eligible alternatives, preventing manual data entry errors.
